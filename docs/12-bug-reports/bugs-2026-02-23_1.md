# mxcli Bug Report — Session 2026-02-23

## BUG-001: DESCRIBE MICROFLOW returns incomplete/sparse MDL

**Severity:** High
**Area:** DESCRIBE

`describe microflow` returns only a skeletal stub (single IF with empty body, no RETURN) even for microflows with 10–28 catalog activities. The output is not usable for round-tripping or auditing logic.

**Repro:**
```sql
describe microflow FeedbackModule.VAL_Feedback;
```

**Expected:** Full MDL representation of all activities.
**Actual:** ~4 lines with an empty IF body and no RETURN statement, despite the catalog reporting 28 activities.

## BUG-002: FOLDER clause not valid on CREATE SNIPPET

**Severity:** Medium
**Area:** MDL Parser

`folder` is accepted syntactically but rejected at execution for snippets. This is inconsistent with microflows (where `folder` is a standalone keyword) and pages (where `folder:` appears inside the `()` block).

**Repro:**
```mdl
create snippet Summit.NavigationMenu folder 'Navigation'
{
  ...
}
/
```

**Error:**
```
mismatched input 'FOLDER' expecting {'(', '{'}
```

**Expected:** Either support `folder` on snippets (consistent with microflows), or produce a clearer error message indicating FOLDER is not supported for this object type.

---

## BUG-003: RETRIEVE WHERE rejects XPath bracket notation

**Severity:** Medium
**Area:** MDL Parser

Square-bracket XPath notation (`where [association = $Var]`) — which is standard in Mendix microflow XPath — is rejected by the MDL parser. Only bare expressions are accepted.

**Repro:**
```mdl
retrieve $Items from Summit.OrderItem
  where [Summit.OrderItem_Order = $Order];
```

**Error:**
```
extraneous input '[' expecting ...
```

**Workaround:**
```mdl
retrieve $Items from Summit.OrderItem
  where Summit.OrderItem_Order = $Order;
```

**Expected:** Either support bracket notation (matching Mendix XPath conventions), or document the difference clearly in the MDL reference.

## BUG-004: ALTER PAGE REPLACE silently drops pluggable widget Binds

**Severity:** High
**Area:** ALTER PAGE

When replacing a widget using `alter page ... replace ... with { ... }`, the `binds` property of pluggable input widgets (e.g., `radiobuttons`) is not persisted. The replacement completes without error, but the binding is missing, causing a subsequent MxBuild failure.

**Repro:**
```mdl
alter page Summit.Customer_NewEdit {
  replace cbCreditRating with {
    radiobuttons rbCreditRating (label: 'Credit Rating', binds: CreditRating)
  }
};
```

**MxBuild error:**
```
Property 'Value' of radio buttons 'rbCreditRating': No value selection has been made
```

**Workaround:** Use `create or replace page` to rewrite the entire page.

**Expected:** `binds` should be persisted after a REPLACE operation, the same as all other widget properties.

## BUG-005: NOT NULL ERROR constraint fires at CREATE time, not at commit

**Severity:** Medium
**Area:** Runtime

When an entity attribute has `not null error 'message'`, creating an object with an empty string value (`Name = ''`) immediately triggers the validation error dialog — before the edit popup even opens. The constraint should fire at commit/save time, not at object instantiation.

**Repro:**

Entity definition:
```mdl
create persistent entity Summit.Customer (
  Name: string(200) not null error 'Customer name is required',
  ...
);
```

Microflow:
```mdl
$NewCustomer = create Summit.Customer (Name = '');
show page Summit.Customer_NewEdit ($Customer = $NewCustomer);
```

**Result:** Error dialog "Customer name is required" fires before the page is shown.

**Workaround:** Initialize with a placeholder value (`Name = 'New Customer'`).

**Expected:** `not null error` should behave as a commit-time validation, consistent with how validation feedback works in standard Mendix microflows.

## BUG-006: Multi-statement heredoc pipe to REPL silently discards statements

**Severity:** Low
**Area:** REPL

Piping multiple MDL statements to the mxcli REPL via a heredoc produces no output and no execution — statements are silently dropped with no error.

**Repro:**
```bash
cat <<'EOF' | ./mxcli
connect local 'MxSummit.mpr';
create or replace page Summit.Customer_NewEdit (...) { ... } /
EOF
```

**Result:** No output, no error, no execution.

**Workaround:** Write statements to a `.mdl` file and use `execute script 'file.mdl';` from the REPL.

**Expected:** Piping statements should work the same as typing them interactively, or produce a clear error if stdin mode is not supported.
